01
Typography

文本、字体与可复制性

这一节优先展示 dompdf.js 最值得看的能力：文字仍然是文字，而不是被摊平成整页位图。对业务文档来说，可检索、可复制、可缩放通常比“看起来差不多”更重要。

双语业务摘要

城市更新试点 2026 / Urban Renewal Pilot 目前进入交付前评审阶段。当前版本重点验证多语言段落在 A4 分页、字体嵌入、字宽计算与表格复制中的稳定性，确保导出后的 PDF 能够满足“归档、分发、审阅、检索”四类业务场景。

For the export engine, a realistic stress sample matters more than decorative gradients. The document should contain long paragraphs, dense tables, repeated labels, page boundaries, and inline vector figures so the resulting PDF exposes actual trade-offs in text fidelity, object count, and payload size.

这里故意放较长、较密的正文文本，让 html2pdf.js 的逐页位图输出更容易在体积上放大，而 dompdf.js 仍然保持以文本和矢量对象为主的输出结构。

本页重点验证

  • 中文、英文、数字与标点连续可复制
  • 字体嵌入后仍能搜索关键字和项目编号
  • 多页导出时页眉页脚与正文不互相覆盖
  • 结构化表格和列表不因缩放产生模糊
  • 复杂容器可通过 divisionDisable 保持整体跨页

Pagination

分页压力测试与附录

这部分故意堆叠较长的正文和多条记录，让输出自然跨页。对于 html2pdf.js 来说，每一页更像一张大截图；对于 dompdf.js 来说，文本密度增加并不会线性放大到同样程度。

下列条目采用重复但不完全相同的业务文本，用来验证长列表分页、连续复制以及页边界附近的段落切分。

    强制分页样本

    Phase 1 · Snapshot Contract

    冻结 DOM Snapshot 字段，确保文本、图片、布局与分页控制信号都能稳定进入 Worker。

    Phase 2 · Typography Validation

    重点回归中文字体、数字、英文单词、代码片段、数学符号与复制顺序。

    Phase 3 · Document Scale

    用真实长度的文档内容拉开最终 PDF 体积差，并确认分页切点不漂移。

    divisionDisable 样本

    这个模块带有 divisionDisable 属性。跨页时它应该整体移动到下一页，而不是被从中间切断。真实业务里，签字区、报价摘要、审批意见、票据封面等模块通常都需要这种保护。

    如果这个卡片在页边界处被截开，说明复杂容器在分页合并和块级迁移上的策略还需要继续修正。
