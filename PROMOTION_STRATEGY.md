# dompdf.js 宣传策略与角度分析

> 面向 TRAE AI 创造力大赛（学习工作赛道，库/SDK 品类）及其他推广场景

---

## 一、核心定位：一句话说清楚你是谁

**"唯一能在浏览器端生成「真矢量可编辑 PDF」的纯前端库"**

传统方案（html2canvas + jsPDF）的本质是「截图→贴图」，生成的是**图片 PDF**（文字不可选、不可搜索、锯齿感严重、文件体积大）。dompdf.js 读取 DOM + CSS 后将每个元素直接绘制为 PDF 原生指令（文字就是文字、线条就是线条），产出的是**真正的矢量 PDF**。

## 二、宣传故事线（4 条主线，选 1-2 条主推）

### 🅰 故事线一：「技术突破」—— 为什么别人做不到，而你做到了？

**适合：技术社区（掘金/知乎/Reddit/HackerNews）、TRAE 评审**

| 对比维度 | html2canvas + jsPDF | dompdf.js |
|---------|---------------------|-----------|
| 输出类型 | 图片嵌在 PDF 里 | 真矢量 PDF |
| 文字可选/可搜索 | ❌ | ✅ |
| 文件体积（同内容） | 大（图片） | 小（矢量指令） |
| 放大后清晰度 | 模糊/锯齿 | 无损 |
| 数千页支持 | 受 canvas 高度限制 | ✅ 无限制 |
| 服务端依赖 | 无 | 无 |

**核心叙事**：html2canvas 诞生 10 年，npm 周下载量 190 万+，但所有人都忍受着「截图转 PDF」的低质量输出。dompdf.js 通过**改造 html2canvas 的 canvas-renderer 为 pdf-renderer**，绕过了 canvas 位图渲染的限制，直接用 jsPDF API 绘制 PDF 原生元素。这是一个「站在巨人肩膀上但走了一条完全不同的路」的技术故事。

**金句**：
> "html2canvas 把网页变成图片，dompdf.js 把网页变成真正的 PDF。"

### 🅱 故事线二：「刚需场景」—— 谁在痛？痛到什么程度？

**适合：产品宣传、Landing Page、演示视频**

#### 场景 1：电子发票 / 电子合同 / 电子病历

- **痛点**：ERP/OA/医疗系统需要在浏览器端生成可打印、可归档的 PDF 文档
- **现状**：要么用图片 PDF（甲方不认），要么走后端渲染（增加服务器成本 + 延迟）
- **dompdf.js 解法**：纯前端生成矢量 PDF，含页码、页眉页脚、签章区域，文件小可直接归档

#### 场景 2：在线报表 / 数据看板导出

- **痛点**：BI 看板导出时文字模糊、图表锯齿、文件几十 MB
- **dompdf.js 解法**：矢量化渲染 + 分页支持，千页报表轻松导出

#### 场景 3：简历生成器 / 证书生成

- **痛点**：前端生成的 PDF 质量差，HR 打开文字模糊
- **dompdf.js 解法**：矢量文字 + 自定义字体（含中文字体），导出的简历可直接被 ATS 解析

#### 场景 4：国际化 SaaS 产品

- **痛点**：中/日/韩/阿拉伯等多语言混合排版，字体乱码
- **dompdf.js 解法**：`langFontConfig` 按 Unicode 码位自动匹配字体，一个配置搞定多语言

**叙事框架**：先展示场景痛点的「Before」（截图画质差/文字模糊/文件大）→ 再展示 dompdf.js 的「After」（清晰锐利/体积小/可编辑）→ 对比冲击。

### 🅲 故事线三：「从 0 到 1 的独立构建」

**适合：TRAE 大赛评审、个人品牌、GitHub 故事**

dompdf.js 不是 html2canvas 的 fork/plugin，而是：
- 保留了 html2canvas 的 DOM 解析层（成熟稳定）
- **完全重写**了渲染层：从 Canvas API 调用 → jsPDF API 调用
- 自研了**分页引擎**（`paginate.ts`，280 行核心算法）
- 自研了**多语言字体匹配引擎**（`langFontConfig`，按 Unicode 码位自动选择字体）
- 支持 PDF 加密、压缩、仅嵌入已用字体等专业特性

这是一个理解底层原理后「推翻重建」的项目，不是「调包侠」。

**差异化优势一句话**：比纯前端方案输出质量高一个档次，比服务端方案部署简单一个数量级。

## 六、竞争力护城河

这些是别人难以快速复制的壁垒，宣传时可强调：

1. **分页引擎**：不是简单截断，而是 DOM 树级别的智能分页（`divisionDisable`/`pageBreak` 属性控制 + 页眉页脚动态计算）
2. **langFontConfig 字体匹配**：按 Unicode 码位逐字匹配字体，支持中日韩阿拉伯语混合排版，这是 jspdf 生态的独特功能
3. **html2canvas 兼容层**：API 设计兼容 html2canvas 用户习惯，降低迁移成本
4. **foreignObjectRendering 降级方案**：对无法矢量化的复杂 CSS（box-shadow、复杂渐变）自动降级为 SVG foreignObject 渲染，保证 100% 覆盖率
